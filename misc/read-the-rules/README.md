# Nom du défi

**`Erreur-404`** [](https://github.com/Erreur-404)

## Description FR

Il est toujours important de lire les règles. Si tu l'as fait, tu sauras résoudre ce défi.

```
## Règles
- En participant à la compétition vous consentez aux règles suivantes et vous vous engagez à les respecter.
- Tout nom d’utilisateur déplacé, injurieux, irrespectueux ou pouvant porter préjudice est proscrit.
- Soyez courtois et respectueux.
- Il est interdit d’attaquer scoreboard de manière générale.
- Les attaques DoS/DDoS contre l'infrastructure et/ou les défis sont strictement interdites.
- Tout déni de service est interdit.
- Le brute-force des formulaires de soumission des flags est interdit.
- Le brute-force contre les défis n'est pas permis (e.g., brute-force de login, URL path, etc.), sauf si explicitement permis dans la description du défi.
- L'utilisation d'outils d'exploitation automatique et de scanners de vulnérabilité de masse est strictement interdite, sauf si explicitement permis dans la description du défi.
- Le partage de flags ou des solutions avant la fin de la compétition est interdit.
- Si quelque chose semble brisé, laissez le savoir aux modérateurs sur Discord.
- Il est obligatoire de placer “rules” à l'intérieur des accolades du format proposé plus bas pour valider le premier défi.
- L’accumulation de flags est interdit. Vous devez soumettre les flags aussitôt que vous les trouvez.
- L’équipe de modération se réserve le droit de disqualifier toute personne susceptible de violer les règles.

## Informations générales

- Tous les flags suivent l'expression régulière suivante, sauf si mentionné autrement: ^polycyber{.+}$. Par exemple, ceci serait un flag valide: polycyber{fake_flag}.
- Les défis seront publiés par vagues, donc même si tu finis tous les défis, reste à l'affût de la publication d'autres défis !
- Toute information critique concernant la compétition sera communiquée via Discord et/ou la plateforme CTFd.
- Le scoreboard sera invisible au courant des 30 dernières minutes.
```

## Description EN

It's always important to read the rules. If you did, you'll know how to solve this challenge.

```
## Rules
- By participating in the competition, you agree to the following rules and commit to abiding by them.
- Any username that is inappropriate, offensive, disrespectful, or potentially harmful is prohibited.
- Be courteous and respectful.
- It is forbidden to attack the scoreboard in any way.
- DoS/DDoS attacks against the infrastructure and/or challenges are strictly prohibited.
- Any form of denial of service is prohibited.
- Brute-forcing submission flag forms is prohibited.
- Brute-forcing against challenges is not allowed (e.g., login brute-force, URL path, etc.), unless explicitly permitted in the challenge description.
- The use of automated exploitation tools and mass vulnerability scanners is strictly prohibited, unless explicitly permitted in the challenge description.
- Sharing flags or solutions before the end of the competition is prohibited.
- If something appears broken, let the moderators know on Discord.
- It is mandatory to place "rules" inside the curly braces of the format proposed below to validate the first challenge.
- Flag accumulation is prohibited. You must submit flags as soon as you find them.
- The moderation team reserves the right to disqualify anyone suspected of violating the rules.

## General Information

- All flags follow the following regular expression, unless stated otherwise: ^polycyber{.+}$. For example, this would be a valid flag: polycyber{fake_flag}.
- Challenges will be released in waves, so even if you finish all challenges, stay alert for the publication of additional challenges!
- Any critical information regarding the competition will be communicated via Discord and/or the CTFd platform.
- The scoreboard will be invisible during the last 30 minutes.
```
